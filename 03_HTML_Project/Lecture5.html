Why Semantic HTML?

Semantic tags give meaning to content. They improve accessibility, SEO, and code readability.

Common Semantic Elements

  • <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>.
  • Use <figure> and <figcaption> for images with captions.

Accessibility Basics

  • Always provide alt for images.
  • Use <label> for form fields and connect with for attribute when using id.
  • Provide logical document order and heading structure for screen readers.
  • Use aria-* attributes only when semantic HTML cannot express the meaning.

Example: Accessible form snippet

<form>
  <label for="email">Email:</label>
  <input id="email" name="email" type="email"><br>
  <label><input type="checkbox" name="subscribe"> Subscribe to updates</label>
</form>

Practice Tasks

  1. Convert a non-semantic page that uses many <div>s into semantic tags.
  2. Add accessible labels and alt text to an existing form and image list.
