How the Internet Works (Concise Notes)

Principles (Short)

  1. Need to share information: connect two computers with a link (electrical/light signals).
  2. Scale problem: use a switch for many devices (local network).
  3. Connecting networks: use routers to connect different local networks.
  4. Addressing: IP addresses uniquely identify devices.
  5. Reliability: break data into packets and use protocols (TCP/IP).

Important Terms

IP Address (IPv4)
32-bit address shown as four decimals (e.g., 172.217.16.142). Limit ≈ 4.3 billion addresses.
Public IP
Globally unique address assigned by ISP; your router usually has one.
Private IP
Used inside local network (e.g., 192.168.x.x). Only unique inside that network.
IPv6
128-bit addressing (hexadecimal groups) created to solve IPv4 exhaustion.
MAC Address
Hardware-level 48-bit identifier for network interfaces (e.g., 3C:22:FB:A3:B4:C5).
Port Number
16-bit number (0–65535) that identifies services on a host (HTTP:80, HTTPS:443).
DNS
Translates domain names to IP addresses (root → TLD → authoritative server → result).

Quick Examples / Commands

Windows: ipconfig /all or getmac /v

macOS / Linux: ifconfig

Summary

The internet is a network of networks. Routers move packets using IP addresses. DNS maps human names to IPs. Use these fundamentals as mental anchors.
